When would you want to use a remote repository rather than keeping all your work
local?

    When repository is to be shared with others

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    Changes to pull might be too small to update over the network.

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

    Forks are a way of creating a copy of a repository on githubs. Used when changes needs to be made to a repository but without affecting the original 
    Clones are a way of creating a copy of repository from remote to local or local to local. 
    Branches are a way of creating copies of commit histories. Used to test out new feature 

What is the benefit of having a copy of the last known state of the remote
stored locally?

    It allows one to work offline with a copy.
